# Auto-Scan Feature Screenshots

This directory contains screenshots documenting the auto-scan feature implementation for GitHub issue [#116](https://github.com/neonwatty/meme-search/issues/116).

## 📸 Automated Capture

Screenshots are automatically generated using Playwright automation:

```bash
# Prerequisites: Rails test server running on port 3000
npm run capture:screenshots
```

**Runtime**: ~15 minutes to capture all 23 screenshots

## 🛠️ Technical Details

### Automation Script

The screenshots are generated by `/scripts/capture-screenshots.ts` which:
- Seeds test database with paths in various states
- Navigates through all UI workflows
- Captures screenshots at key moments
- Handles dark mode, dropdowns, and flash messages

### Test Data Setup

```typescript
// 6 paths with different auto-scan states:
- example_memes_1: Manual only
- example_memes_2: Up-to-date (scanned 15 min ago, freq: 30 min)
- test_valid_directory: Due now (scanned 70 min ago, freq: 60 min)
- test_empty_directory: Currently scanning
- comics: Failed (with error message)
- memes: First scan pending (never scanned)
```

## 🔄 Re-running Automation

To update screenshots after UI changes:

```bash
# 1. Ensure Rails test server is running
cd meme_search/meme_search_app
mise exec -- bin/rails server -e test

# 2. Run automation (in separate terminal)
npm run capture:screenshots
```

The script will overwrite existing screenshots with fresh captures.
